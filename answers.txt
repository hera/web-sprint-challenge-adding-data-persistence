1. Explain the difference between Relational Databases and SQL.
SQL is a language for making queries. Relational Databases represent data.

2. Why do tables need a primary key?
So that rows in another table could hold a reference to a row in that table. (foreign key)

3. What is the name given to a table column that references the primary key on another table.
Foreign key

4. What do we need in order to have a many to many relationship between two tables.
One more table that contains 2 foreign keys (1 for each table)
